Making internal calls

The goal of this block will be to configure IvozProvider to make internal calls, from a fresh base install described in the previous section.

In order to archieve a call between Alice and Bob, it will be required to do task at the 3 levels described in Platform roles, and that’s why we have divided this block into 3 sections:

  • ‘General Configuration’ block
    • Customized configration of the installation
      • Users Proxy
      • Trunks Proxy
    • Standard global configuratio
      • Application Servers
      • Media relays
      • SIP Domains
    • Emulate Demo brand
      • What does this emaulation mean?
  • ‘Brand Configuration’ block
    • SIP Domain of the company
      • I have no time for a DNS registry
    • Emulate Demo company
  • ‘Company Configration’ block
    • Terminals
    • Extensions
    • Users
  • Configure SIP terminals
